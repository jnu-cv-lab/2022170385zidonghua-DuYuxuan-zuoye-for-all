# Lab14 实验记录：棋盘格相机标定

## 1. 实验目标

本实验使用棋盘格图像完成相机标定流程，目标是理解并实现从棋盘格角点检测到相机内参、畸变参数、外参和重投影误差计算的完整过程。实验不使用真实拍摄图片，而是使用程序生成的严格数学透视投影合成棋盘格图片，从而验证相机标定算法流程。

## 2. 棋盘格信息

- 内角点数量：9×6
- 实际方格数量：10×7
- 方格边长：25 mm
- 棋盘格来源：程序生成的严格数学透视变换合成棋盘格图像
- 图像数量：24 张

## 3. 相机信息

- 相机类型：合成针孔相机模型
- 图像分辨率：1280×960
- 镜头类型：普通镜头模型，加入径向畸变和切向畸变
- 真实内参矩阵：

```text
[[900,   0, 640],
 [  0, 920, 480],
 [  0,   0,   1]]
```

- 真实畸变参数：

```text
D = [0.08, -0.03, 0.001, 0.0005, 0.01]
```

其中 D = [k1, k2, p1, p2, k3]。

## 4. 图片姿态说明

24 张合成图片使用不同外参姿态生成，覆盖中心、左侧、右侧、上方、下方、左上角、右上角、左下角、右下角、近距离、远距离、左右倾斜、上下俯仰、顺时针旋转和逆时针旋转等情况。所有棋盘格都完整保留在图像画面内，边界清晰，适合用于角点检测和标定。

## 5. 程序流程

1. 生成棋盘格：使用固定真实相机内参和畸变参数，设置 24 组不同的旋转向量和平移向量，用 `cv2.projectPoints()` 将棋盘格方格角点投影到图像上，再用 `cv2.fillConvexPoly()` 绘制黑白方格。
2. 检测角点：读取合成图片，转换为灰度图，使用 `cv2.findChessboardCorners()` 检测 9×6 内角点。
3. 亚像素优化：对检测成功的角点使用 `cv2.cornerSubPix()` 进行亚像素级优化。
4. 相机标定：使用 `cv2.calibrateCamera()` 估计相机内参矩阵 K、畸变参数 D，以及每张图片对应的外参。
5. 重投影误差计算：使用 `cv2.projectPoints()` 将三维角点重新投影到图像平面，与检测角点比较，计算每张图片的平均误差。
6. 去畸变处理：使用 `cv2.undistort()` 对至少一张图片进行去畸变，并保存原图与去畸变图的左右对比图。

## 6. 说明

本实验使用合成数据完成流程验证，因此标定结果反映的是合成相机模型，不代表真实手机或真实摄像头参数。真实相机标定时，应使用同一台真实相机在不同姿态和距离下拍摄的清晰棋盘格图片。
